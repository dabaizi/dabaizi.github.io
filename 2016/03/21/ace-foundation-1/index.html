ACE的常用显示代码的诊断输出宏有三个，ACE_DEBUG，ACE_ERROR和ACE_TRACE。除此之外还有ACE_ASSERT、ACE_HEX_DUMP、ACE_RETURN、ACE_ERROR_RETURN、ACE_ERROR_INIT、ACE_ERROR_BREAK……使用语法和C语言里面的printf很类似，语法如下：

其中，level指的就是消息的严重级别，最常用的级别就是LM_DEBUG和LM_ERROR。其他的严重性值如下表所示：

严重级别 含义
LM_TRACE 指示函数调用次序的消息
LM_DEBUG 调试信息
LM_INFO 消息含有通常只在程序调试时使用的信息
LM_NOTICE 不是出错的情况，而是可能需要特别处理的情况
LM_WARNING 警告信息
LM_CRITICAL 紧急情况，比如硬件设备出错
LM_ALERT 应该立即纠正的情况，比如系统数据库损坏
LM_ERROR 错误信息
LM_EMERGENCY 万分紧急的情况，通常会广播给所有用户

而string中则可以包括包括格式转换操作符，就像c语言里面的printf(“i=%d;j=%d”, i, j);d表示int类型一样，ACE中也有相应的指令集，常见的如下表所示：

代码 参数类型 显示
A ACE_timer_t 浮点数，如果平台不支持，则显示长十进制的数字
a — 在显示输出后中止程序
c char 单个字符
C char * 字符串（窄字符）
i.d int 十进制数字
I — 根据从ACE_Trace::get_nesting_indent()获得的嵌套深度缩进输出
e,E,f,F,g,G double 双精度浮点数
l — 日志宏出现之初的行号
M — 消息严重级别的文本形式
m — 与errno值对应的消息，和strerror()所做的事情一样
N — 日志宏所在的文件的名称
n — 传给ACE_Log_Msg::Open()的程序名
o int 八进制数
P — 当前进程的ID
p ACE_TCHAR * 指定的字符串，接着是适当的errno消息，也就是说，perror()所做的事一样
Q ACE_UNIT64 十进制数
r void(*)() 无显示：调用指定的函数
R int 十进制数
S int 编好号的信号的名称
s ACE_TCHAR * 字符串：取决于ACE_TCHAR类型，窄的或者宽的
T — 当前时间，格式是hour:mimute:sec.usec
D — 时间戳，month/day/year hour:minute:sec.usec
t — 调用线程的ID（如果是单线程，就是1）
u int 无符号十进制数
w wchar_t 单个宽字符
W wchar_t * 宽字符串
x,X int 十六进制数字
@ void * 以十六进制表示的指针值
% N/A 单个百分号：”%“
